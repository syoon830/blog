자바스크립트 - 8. 원시 값과 객체의 비교

Tags:
javascript
Date:2024-02-15
데이터 타입은 크게 원시 타입과 객체 타입으로 구분
원시값은 식별자 메모리 주소에 값이 저장되어있고
객체는 식별자 메모리 주소에 해당 객체 메모리 주소가 저장되어 있다

원시 값

원시 타입의 값, 즉 원시 값은 변경 불가능한 값
var score;
score = 80;
score = 90;
undefined가 80으로 바뀌고 80이 90으로 바뀌는게 아니라
값은 새로운 메모리 공간을 확보하고 값을 저장한 후, 변수는 원시 값을 가리킨다.
불변성을 갖는 원시 값을 할당한 변수는 재할당 이외에 변수 값을 변경할 수 있는 방법이 없다.

값에 의한 전달

var score = 80;

// copy 변수에는 score 변수의 값 80이 복사되어 할당된다.
var copy = score;

console.log(score, copy); // 80 80
console.log(score === copy); // true
score 변수와 copy 변수의 값 80은 다른 메모리 공간에 저장된 별개의 값이다.
score = 100;
console.log(score, copy); // 100 80
console.log(score === copy); // false

원시 값인 문자열은 다른 원시 값과 비교할 때 독특한 특징이 있음

문자열은 유사 배열 객체이면서 이터러블이므로 배열과 유사하게 각 문자에 접근할 수 있다.
var str = 'string';

// 배열처럼 인덱스를 사용해 각 문자에 접근할 수 있다.
console.log(str[0]); // s

// 원시 값인 문자열이 객체처럼 동작한다.
console.log(str.length);
console.log(str.toUpperCase());
어떻게 원시 값이 객체처럼 동작할까? 그 이유는 원시 값을 감싸는 래퍼 객체로 자동 변환되기 때문이다. 자세한건 이후에
여기서 퀴즈!
var arr = [1, 2, 3];
arr[0] = 4;
console.log(arr) // ??

var str = 'string';
str[0] = 'S';
console.log(str); // ??

객체

객체 타입의 값, 즉 객체는 변경 가능한 값이다.
용어정리
일반적으로 원시 값을 할당한 경우 “변수는 O값을 갖는다” 또는 “변수의 값은 O다”라고 표현하고
객체를 할당한 변수의 경우 “변수는 객체를 참조하고 있다” 또는 “변수는 객체를 가리키고 있다”라고 표현

참조에 의한 전달

var person = {
    name: 'Lee'
}

var copy = person;

copy.name = 'kim';

console.log(person.name); // kim;
console.log(copy.name); // kim;
정확한 용어는 존재하지 않지만 전달되는 값의 종류에 원시 값인지 참조 값인지 구별해서 강조하는 의미에서 “값에 의한 전달”, “참조에 의한 전달”로 구분하여 부르기 함.
