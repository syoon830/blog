nuxt에서 localstorage 이슈

Tags:
nuxtssrmiddleware
Date:2024-01-16
기존에 cookie에 auth 정보를 저장했을때는 미들웨어가 잘 동작을 했는데, 테스트상 localstorage에 저장을 해보니 미들웨어가 제대로 동작을 안한다.
import { useAuthStore } from "~/store/auth";

export default defineNuxtRouteMiddleware((to, from) => {

    const authStore = useAuthStore();
    const nonAuthPages = ['index', 'login'];
    if (authStore.isAuthenticated) {
        if (nonAuthPages.includes(to.name)) {
            return navigateTo('/dashboard');
        }
    } else {
        if (!nonAuthPages.includes(to.name)) {
            return navigateTo('/login');
        }
    }
}
위 코드를 보면 로그인하면 index, login 페이지 접근 못하게, 로그인 안하면 index, login을 제외한 나머지 페이지 접근 못하게 코드를 짰다.
로그인한 상태에서는 organization으로 접근을 해야하는데 login 페이지를 거쳐서 dashboard로 이동을 한다.
원인을 살펴보니 ssr에서 localStorage에 접근을 못하여 예상하지 못한 로직으로 흐름이 발생한거였다. (server-side에서는 cookie에만 접근이 가능하다.)

localStorage에 auth정보 저장 시 흐름 (isAuthenticated는 true)

1.
organization 페이지에 접근
a.
server-side에서 middleware 동작
i.
이때 server-side에서는 localstorage에 접근을 못하니까 authStore 정보를 못 얻어온다.
ii.
navigateTo(’/login’)이 실행된다.
2.
login 페이지 접근
a.
server-side에서 middleware가 동작
i.
마찬가지로 server-side에서는 localstorage에 접근을 못하니까 authStore 정보를 못 얻어온다.
ii.
navigateTo(’/login’)이 실행된다.
3.
같은 페이지로 리다이렉트 됐기 때문에 더이상 ssr은 실행되지 않고 csr이 실행된다.
4.
login 화면이 렌더링
a.
client-side에서 middleware 동작
i.
csr에서는 localstorage에 접근이 가능하기 때문에 authStore 정보를 얻어옴
ii.
nonAuthPages에 포함됨
iii.
navigateTo(’dashboard’)이 실행된다.
5.
dashboard 화면이 렌더링 (client-side에서 middleware 동작은 생략)
SSR 환경에서의 navigateTo는 먼저 서버 측에서 처리되고 그 후 클라이언트 측에서 렌더링된다. 반면, CSR 환경에서의 navigateTo는 클라이언트 측에서만 처리되며, 서버 측 처리는 발생하지 않는다.
